Why use Seven?

The story of performance.

Blazing Fast

Built on top of Bun. Seven-Discord parses macros 200x faster than traditional interpreted libraries.

Zero Dependencies

Say goodbye to `node_modules` hell. No bloat, no vulnerabilities.

Scale Ready

SecureToken encryption, Rate Limit handling, and Resume capability built-in.

The Seven Story

We were tired of bloated libraries. Discord.js requires too much boilerplate. Other macro libraries were slow and buggy.

So we built Seven. Named after the 7 core principles of software design: Simplicity, Speed, Security, Scalability, Stability, Structure, and Style.

It runs on Bun, utilizing the fastest JavaScript runtime on the planet. It handles Sharding, Intents, and Interactions natively. It is, simply put, the future.

Developer Guide

1. Installation

bunx seven-discord --init --name my-bot

2. Folder Structure

Load commands recursively.

const bot = ...
bot.loadCommands("./commands");

3. Creating Commands

Example commands/ping.ts:

export default {
  name: "ping",
  code: "s.reply[Pong!]"
};

Macro Documentation

Event Reference

Client Events

  • Ready

    Fired when the bot connects.

Guild Events

  • MessageCreate

    Fired when a message is sent.

  • GuildMemberAdd

    Fired when a user joins.

  • GuildMemberRemove

    Fired when a user leaves.

  • InteractionCreate

    Fired on slash command or button click.
